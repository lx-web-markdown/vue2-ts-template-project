Vue 组件化应用构建

  • 组件系统是 Vue 的核心特性之一，允许我们用小型、独立、可复用的组件来构建大型应用。
  • 每个组件本质上是一个拥有预定义选项的 Vue 实例。
  • 组件之间通过 props 进行数据传递，实现父子解耦。
  • 本页演示了如何用 Vue 组件化思想构建一个简单的待办事项列表。
代码示例
<div id="app-component">
  <h2>我的待办事项</h2>
  <ol>
    <todo-item
      v-for="item in groceryList"
      v-bind:todo="item"
      v-bind:key="item.id"
    ></todo-item>
  </ol>
</div>

<script>
// 注册一个全局组件
Vue.component('todo-item', {
  props: ['todo'],
  template: '<li>{{ todo.text }}</li>'
})

var app = new Vue({
  el: '#app-component',
  data: {
    groceryList: [
      { id: 0, text: '蔬菜' },
      { id: 1, text: '奶酪' },
      { id: 2, text: '随便其它什么人吃的东西' }
    ]
  }
})
</script>
实际效果

我的待办事项

详细讲解
  • 我们通过 Vue.component 注册了一个名为 todo-item 的全局组件。
  • 该组件通过 props 接收父组件传递的数据，实现了父子解耦。
  • 在父组件模板中，使用 v-for 遍历 groceryList，为每个待办项渲染一个 todo-item 组件实例。
  • 每个组件实例都通过 v-bind:todo="item" 传递不同的数据。
  • 通过 key 保证每个组件的唯一性，有助于 Vue 内部高效渲染。
  • 组件化让代码更易维护、复用和扩展，是构建大型应用的基础。
在控制台输入：app.groceryList.push({ id: 3, text: '新项目' })，你会发现列表最后添加了一个数据。
